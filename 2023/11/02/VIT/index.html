VIT论文阅读并实现

一、论文简介

1.1 摘要核心

1、Transformer在NLP领域中已经成为了首选模型，但在CV中应用还比较少；

2、在CV中，Attention机制只是作为一个CNN补充或者作为网络中的一部分在使用；

3、我们使用纯Transformer（纯Attention）结构就可以在图像分类任务上取得不错的结果；

4、在足够大的训练集上训练后，ViT可以拿到和CNN的SOTA（state-of-the-art convolutional networks）不相上下的结果。

1.2 研究背景

​ 尽管Transformer结构非常强，但是在视觉任务中的应用比较有限。Transformer提出后在NLP领域中取得了极好的结果，其全Attention的结构，不仅增强了特征提取能力，还保持了并行计算的特点，可以又好又快地完成NLP领域内的几乎所有任务，极大地推动了自然语言处理地发展

​ 但在CV的应用非常有限，此前只有目标检测中的DETR大规模使用了Transformer，别的领域很少，而纯Transformer的则几乎没有。

​ Transformer的优势（相比于CNN）：1、并行计算；2、全局视野；3、灵活的堆叠能力。

​ NLP中的模型Transformer+CV中的基础任务图像分类，需要解决很多问题，如输入不一致等。

1.3 研究成果

​ ViT和ResNet Baseline取得了不相上下的结果。

​ ImageNet等：数据集名称

​ JFT：谷歌私有数据集，规模为ImageNet的20~30倍

​ ViT-H/14：ViT-Huge模型，输入序列14x14

1.4 研究意义

​ 展示了在计算机视觉中使用纯Transformer结构的可能；

​ 拉开了Transformer研究热潮的序幕。

二、论文详解

2.1 模型总览

​ 因为Transformer在NLP中很好用，所以ViT结构在进行设计时希望可以直接把Transformer应用于图像领域，所以对其作了最小的改动。具体而言，就是把一张图片切片，然后将其当作一个序列，加上线性的embedding后放入到Transformer中去。类比于NLP，此处的Image patch（图像切片）是作为token输入，即每一个切片都是一个处理单元，如下图所示。

​ 将图像处理为序列后输入进入Transformer Encoder，该部分由L个图右所示组件组成，其中Norm用于标准化，MLP为两层FC层。模型中有三个部分我不是很理解，分别为①Multi-Head Attention；②Position Embedding；③Linear Projection of Flattened Patches，接下来逐一解读这三个部分。

2.2 Multi-head attention

​ 为了理解Multi-head attention，需要先了解一下Attention的机制。Transformer有两个部分，分别是编码器和解码器，编码器用于将输入的内容编码成“意思”（即该语言所指的东西），解码器用于将“意思”映射为目标语言中对应的表示方式。但是人工在进行翻译时有这样一种直觉：一个翻译只与有限的几个词有关，因此就在Transformer中引入了attention机制，对于需要关注的词，我们给一个更高的权重，不需要关注的词，给的权重则比较低，attention本质上是对token进行加权平均。attention需要考虑的问题是如何得到相应的权重，一般使用相似度计算来解决。

​ Multi-head attention用到了多个Self Attention。Self Attention的计算用到了Scaled dot product，计算公式为
$$
Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})
$$
​ 输入时，我们会将原本的对象进行转换，如将单词转换为词向量，图像转变为序列等。公式中的Q、K、V都是序列或词向量进行不同的矩阵乘法后得到的，其中Q是Query，用于对输入进行查询；Key为键值，通过与查询的值进行相似度计算（使用点积）获得相关度最高的关键字；V为价值，根据关键字返回对应的数值。本质是对input进行了线性变换，可以使用nn.Linear实现这些可学习的矩阵。$\sqrt{d_k}$为规模，一般取64，除以该值进行缩放，可以避免出现较大的数值，防止因softmax产生的极端值导致的梯度消失。最后使用softmax计算权值，每一个q与所有k计算得到的相似度之和为1(如q1和k1,k2,…,kn计算得到的相似度之和为1)。Self Attention实际上是在计算每个q和每个k的相似度。得到相似度后，将其作为权重与$k_n$对应的值$v_n$进行相乘并相加，最终得到输入$x$的sum结果$z$，代表了单词意思到句中意思的转化。直接将向量合并为矩阵可以进行并行计算。

​ MultiHead Attention就是有多层Self Attention Layer，有多个$w_q$,$w_k$,$w_v$，将上述操作重复多次，每多一层Self Attention Layer就会多得到一组$z$，最后将得到的结果concat在一起，经过一层Linear层得到最后的结果。

​ 目的是为了给注意力提供多种可能性。不同头的注意力可以集中到一个事物的不同地方上去。

2.3 输入端适配

​ 本小节解读图像如何转换成transformer能够接受的输入格式，分为两个部分，第一个部分为图片是如何切分的，第二个部分为position encoding是如何操作的。

​ Transformer的输入要求是一个序列，对于我们输入的一个原始图像，首先进行切图重排，将原图切分成3*3共9个小块，每一小份叫做一个patch；接着对每一个patch做flatten操作，即将原本3维的张量拉成1维的向量，将每一个拉平的patch称为flatten patch；然后对flatten patch做Linear projection（使用fc），即使向量通过全连接层映射为另一个向量，这样就得到了我们需要的输入向量。

​ 第二个部分为position encoding（位置编码）。由于图像切分重排后失去了位置信息，并且Transformer的内部运算是空间信息无关的，所以需要把位置信息编码重新传进网络。ViT使用了一个可学习的Vector来编码，编码vector和patch vector直接相加组成输入。但是不同的input size和patch size可能会得到不同数量的patch，所以编号的方法需要缩放。

​ 经过position encoding编号的输入，除了我们得到的输入向量编号1-9以外，还会有一个编号为0的patch（本质是dynamic pooling layer）用于整合信息，最后使用position0来进行分类。

2.4 实验设置

​ 论文使用了3个不同规模的模型来进行实验，分别为Base、Large和Huge。其中Layers指标代表有多少个Transformer Encoder，hidden size代表embedding进去时候的channel数，MLP size是Encoder中channel数放大的size，Heads数代表有多少个attention head，Params为模型大小，可以看到ViT模型普遍都比较大。

​ 训练时该模型大规模使用Pre-Train。现在大数据集上预训练，然后到小数据集上Fine Tune。迁移过去后，需要把原本的MLP Head换掉，换成对应类别数的FC层，处理不同尺寸输入的时候需要对Positional Encoding的结果进行插值（缩放编号）。

​ 从原论文p7的图像可以看出，ViT的性能需要大量的数据来保证，否则无法充分发挥性能。

3.1 ViT参数

使用了https://github.com/lucidrains/vit-pytorch 的库来进行解读。

Usage

  • image_size: int.
    Image size. If you have rectangular images, make sure your image size is the maximum of the width and height
  • patch_size: int.
    Size of patches. image_size must be divisible by patch_size.
    The number of patches is: n = (image_size // patch_size) ** 2 and n must be greater than 16.
  • num_classes: int.
    Number of classes to classify.
  • dim: int.
    Last dimension of output tensor after linear transformation nn.Linear(..., dim).
  • depth: int.
    Number of Transformer blocks.
  • heads: int.
    Number of heads in Multi-head Attention layer.
  • mlp_dim: int.
    Dimension of the MLP (FeedForward) layer.
  • channels: int, default 3.
    Number of image’s channels.
  • dropout: float between [0, 1], default 0..
    Dropout rate.
  • emb_dropout: float between [0, 1], default 0.
    Embedding dropout rate.
  • pool: string, either cls token pooling or mean pooling

3.2 输入端适配

图像切分重排

​ 其中Rearrange函数来自于einops库，einops是一个用来高效实现元素重排的库，主要用来代替一些需要集成使用reshape，repeat，expand，view，stack，permute才能实现的方法，恰好图片的切分重排就是这样的一个事情。通过b c (h p1) (w p2) -> b (h w) (p1 p2 c)把原本的shape变为想要的shape。

​ 原本的shape为(b,c,h,w)，rearrange变化前将h拆为了h,p1,w拆为了w,p2,所以变化前形状为(b,c,h,p1,w,p2)，这个五元组中，b和c仍然为batchsize和channel，p1为每一个patch的高度，h为图片在纵坐标被分割的块数，p2为patch的宽度，w为图片在横坐标被分割的块数，即完成了把图拆开的工作。变化后为(b,h*w,p1*p2*c)的形状，即将patches拉平，最后经过一个Linear层，得到input vector。

​ 官方演示：https://github.com/arogozhnikov/einops/blob/master/docs/1-einops-basics.ipynb。

用Conv2d实现图像切分重排

​ 能够这样写的原因是MLP和Conv2d的计算过程很像。具体而言，MLP一般处理一维vector，方法为矩阵乘法，也就是n个点积，先乘再加。Conv2d处理二维，方式为卷积，也就是随着窗口滑动的先乘再加。MLP和Conv2d最重要的两点区别为局部连接、权重共享。可以发现在图像切分重排的这个过程中，所有的操作都在每个patch内（局部连接），而对于每个patch的操作是相同的（权重共享），因此可以用conv代替。
